# 파RAG학기 — 시스템 동작방식 & 핵심 알고리즘

---

## 시스템 동작방식

```
① 사용자 입력 (주제 + 기간 + 개인정보)
        ↓
② RRF 하이브리드 검색 (벡터 + BM25 + 날짜 필터)
        ↓
③ 계획서 별도 조회 (page_id 필터, k=20)
        ↓
④ 컨텍스트 조합 → LLM 스트리밍 호출
        ↓
⑤ 보고서 마크다운 스트리밍 출력
        ↓
⑥ (선택) 용어 해설 / 학과 맞춤 재생성 / Word·PDF 다운로드
```

---

## 핵심 알고리즘 6가지

### 1. Notion 인제스천 파이프라인

- **왜 핵심인가**
  - 헤더 기준으로 섹션을 먼저 나눠야 "어느 섹션의 내용인지"가 메타데이터로 보존돼 검색 정밀도가 높아짐
  - 이미지를 텍스트로 변환하지 않으면 검색에서 완전히 누락됨
- **어떻게 사용되는가**
  - `build_vectordb.py` 최초 1회 실행 시 동작, 이후 모든 보고서 생성이 이 결과를 조회

```
┌──────────────┐
│  Notion API  │
└──────┬───────┘
       │
       ▼
┌──────────────────────┐
│  페이지 · DB 수집     │
└──────┬───────────────┘
       │
       ▼
┌──────────────────────────────────────┐
│  MarkdownHeaderTextSplitter          │
│  h1 / h2 / h3 기준 섹션 경계 분리    │
└──────┬───────────────────────────────┘
       │
       ▼
┌──────────────────────────────────────┐
│  RecursiveCharacterTextSplitter      │
│  500자 단위 청크 분할                 │
└──────┬───────────────────────────────┘
       │
       ├─── [Image: path] 마커 있을 때 ──▶ GPT-4o Vision
       │                                    한국어 설명 생성
       │                                         │
       │◀────────────────────────────────────────┘
       │      마커 → 설명 텍스트 인라인 교체
       ▼
┌──────────────────────────────────────┐
│  OpenAI Embedding                    │
│  text-embedding-3-large              │
└──────┬───────────────────────────────┘
       │
       ▼
┌──────────────────────────────────────┐
│  Qdrant 저장 (코사인 유사도)          │
└──────────────────────────────────────┘
```

### 2. RRF 하이브리드 검색

- **왜 핵심인가**
  - 벡터 검색은 의미 유사도에 강하지만 고유명사·날짜·수치 같은 정확한 키워드 매칭에 취약
  - BM25는 키워드에 강하지만 문맥을 이해하지 못함
  - RRF로 두 순위를 합산해 서로의 약점을 보완
- **어떻게 사용되는가**
  - 보고서 생성 요청마다 `_rrf_search()` 호출, 날짜 범위가 있으면 `DatetimeRange` 필터 함께 적용

```
          쿼리 (query)
               │
    ┌──────────┴──────────┐
    │                     │
    ▼                     ▼
벡터 검색              같은 후보 풀에서
similarity_search      BM25Okapi.get_scores()
k × 3 후보 풀 생성     키워드 기반 재순위
    │                     │
    └──────────┬──────────┘
               │
               ▼
        RRF 점수 합산
  1/(60+vec_rank) + 1/(60+bm25_rank)
               │
               ▼
           Top-K 반환
    (날짜 범위 있으면 DatetimeRange 필터 선적용)
```

### 3. 구조화 보고서 생성 (스트리밍)

- **왜 핵심인가**
  - 형식 지시 없이 LLM에게 맡기면 매번 구조가 달라짐
  - 사용자가 입력한 팀명·학번·성명을 LLM이 임의로 바꾸지 못하도록 명시적으로 고정해야 함
- **어떻게 사용되는가**
  - 채팅창에 주제 입력 시 즉시 동작, 결과는 토큰 단위 SSE 스트리밍으로 화면에 실시간 출력

```
┌──────────────────────────────────────────┐
│  사용자 입력 (주제 + 기간 + 팀정보)       │
└──────┬───────────────────────────────────┘
       │
       ▼
  ┌────┴────────────────────┐
  │  RRF 검색 → 관련 청크   │
  │  계획서 조회 (k=20)     │
  └────┬────────────────────┘
       │
       ▼
┌──────────────────────────────────────────┐
│  프롬프트 조합                            │
│  System: 보고서 형식 고정                 │
│    (기본정보 / 계획 표 / 활동 표 / 배운점) │
│  User: 검색 컨텍스트 + 팀명/학번/성명 고정 │
└──────┬───────────────────────────────────┘
       │
       ▼
┌──────────────────────────────────────────┐
│  LLM (temperature=0)                     │
│  chain = REPORT_PROMPT | LLM | Parser    │
└──────┬───────────────────────────────────┘
       │
       ▼
┌──────────────────────────────────────────┐
│  SSE 스트리밍 → 브라우저 실시간 출력      │
└──────────────────────────────────────────┘
```

### 4. 학과별 맞춤 재생성

- **왜 핵심인가**
  - 같은 내용이라도 학과마다 요구하는 표현 방식이 다름 (컴퓨터공학과 vs 경영학과)
  - YAML로 톤·비유·금지 표현을 정의하면 코드 수정 없이 새 학과를 추가할 수 있음
- **어떻게 사용되는가**
  - 보고서 생성 후 카드의 학과 드롭다운에서 선택하면 원본 보고서 + 학과 컨텍스트로 재생성 요청

```
┌──────────────────────────────────────┐
│  학과 선택 (드롭다운)                 │
└──────┬───────────────────────────────┘
       │
       ▼
┌──────────────────────────────────────┐
│  config/departments/{id}.yaml 로드   │
└──────┬───────────────────────────────┘
       │
       ▼
┌──────────────────────────────────────┐
│  컨텍스트 추출                        │
│  • tone            표현 톤·문체       │
│  • terminology     학과 전용 용어     │
│  • analogy_examples 비유 예시         │
│  • avoid           사용 금지 표현     │
└──────┬───────────────────────────────┘
       │
       ▼
  원본 보고서 + 학과 컨텍스트
       │
       ▼
┌──────────────────────────────────────┐
│  LLM 스트리밍 재생성                  │
└──────────────────────────────────────┘
```

### 5. 용어 해설 자동 생성

- **왜 핵심인가**
  - AI가 생성한 보고서에는 전문 용어가 자연스럽게 포함되는데, 비전공자는 이해하기 어려움
  - 본문 강조 + 하단 해설 표로 보고서 완성도를 높이고 검토·수정 유도에도 기여
- **어떻게 사용되는가**
  - 보고서 카드의 "용어 해설 추가" 체크박스를 켜면 실행, 용어가 없으면 원본 그대로 반환

```
┌──────────────────────────────────────┐
│  완성된 보고서 마크다운               │
└──────┬───────────────────────────────┘
       │
       ▼
┌──────────────────────────────────────┐
│  LLM → 전문 용어 최대 10개 추출      │
│  {"terms": [{"term":..,"explanation":..}]}  │
└──────┬───────────────────────────────┘
       │
       ▼
  줄 단위 순회 (헤딩 # · 표 | · 코드블록 ``` 제외)
       │
       ▼
┌──────────────────────────────────────┐
│  첫 등장 위치에 span 삽입            │
│  <span style="color:#e74c3c">용어</span> │
└──────┬───────────────────────────────┘
       │
       ▼
┌──────────────────────────────────────┐
│  보고서 하단 용어 해설 표 추가        │
│  | 용어 | 설명 |                     │
└──────────────────────────────────────┘
```

### 6. 멀티모달 벡터 DB 생성

- **왜 핵심인가**
  - Notion 회의록에는 캡처 화면·그래프·다이어그램이 자주 포함되지만 이미지는 벡터 DB에 직접 저장 불가
  - 이미지를 무시하면 해당 내용이 검색에서 완전히 누락됨
  - Vision 모델로 이미지를 텍스트 설명으로 변환해 청크에 인라인 삽입하면 이미지 내용으로도 검색 가능
- **어떻게 사용되는가**
  - `build_vectordb.py` 실행 시 인제스천 파이프라인 내에서 자동으로 동작, 이후 모든 RAG 검색에 반영

```
┌──────────────────────────────────────────────────┐
│  [1단계] notion.py — 이미지 수집                  │
│                                                  │
│  blocks.children 순회                            │
│       │                                          │
│       ├─ type="image" 감지                       │
│       │       │                                  │
│       │       ▼                                  │
│       │  CDN URL → 로컬 다운로드                  │
│       │  notion_images/{page_id}_{block_id}.png  │
│       │       │                                  │
│       │       ▼                                  │
│       └─ [Image: path] 마커로 청크에 삽입         │
│          메타데이터(block_id/url/caption) 수집     │
└──────────────────────┬───────────────────────────┘
                       │
                       ▼
┌──────────────────────────────────────────────────┐
│  [2단계] vision.py + chunker.py — Vision 설명    │
│                                                  │
│  청크 내 [Image: path] 정규식 감지                │
│       │                                          │
│       ▼                                          │
│  GPT-4o Vision 호출                              │
│  (base64 이미지 + 페이지/섹션 제목 + 앞뒤 200자)  │
│       │                                          │
│       ▼                                          │
│  2~4문장 한국어 설명 생성                         │
│  마커 → 설명 텍스트 교체 → combined_text 완성     │
└──────────────────────┬───────────────────────────┘
                       │
                       ▼
┌──────────────────────────────────────────────────┐
│  [3단계] store.py — 멀티모달 임베딩 & 저장        │
│                                                  │
│  combined_text (본문 + 이미지 설명)               │
│       │                                          │
│       ▼                                          │
│  OpenAI Embedding → 벡터화                       │
│       │                                          │
│       ▼                                          │
│  Qdrant 저장                                     │
│  payload: image_paths / image_descriptions /     │
│           images_json                            │
└──────────────────────────────────────────────────┘
```
